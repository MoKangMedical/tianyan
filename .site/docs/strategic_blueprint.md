## 一、战略定位：不是Aaru中国版，是"商业决策操作系统"

### ❌ 错误定位
"天眼 = Aaru的中国版，做商业预测的"

### ✅ 正确定位
**天眼 = 中国商业决策的操作系统**

类比：
- Aaru卖的是"预测工具" → 天眼卖的是"决策能力"
- 不是替代尼尔森/艾瑞的调研服务 → 是消灭"需要做调研"这件事本身
- 不是AI公司的竞品 → 是所有做决策的人的底层设施

### 定位公式

```
天眼 = 调研行业的 Salesforce
     = 商业决策的 操作系统
     = 合成人群的 AWS
```

**一句话叙事（对外）**：
> "每一家中国企业的每一个商业决策，背后都应该有一个合成人群在替它试错。"

## 二、品类创造：定义"合成人群经济学"

### 不竞争，创造新品类

Aaru已经在"AI预测"品类里占位了。跟它比，你永远是"中国版Aaru"。
**必须创造新品类，让Aaru变成"美国版天眼"。**

### 新品类定义

```
品类名：合成人群经济学 (Synthetic Population Economics)
定义：用AI生成的合成人群替代真人进行商业决策验证的经济学体系
```

### 品类宣言

> **传统商业决策的悖论**：
> - 调研花3个月，市场已经变了
> - 焦点小组8个人，代表不了14亿人
> - 数据报告100页，CEO只看第一页的结论
>
> **合成人群经济学的答案**：
> - 72小时模拟一个10万人的市场反应
> - 不是抽样，是"全量仿真"
> - 不出报告，直接给决策建议

### 品类心智占领

| 对手的品类 | 天眼的新品类 |
|-----------|-------------|
| 市场调研 | 合成人群仿真 |
| 数据分析 | 决策操作系统 |
| AI预测 | 商业数字孪生 |
| 咨询报告 | 自动决策引擎 |

## 三、战略杠杆：四根撬动市场的杠杆

### 杠杆1：政策红利 — "数字经济"国策

中国"十四五"明确：
- 数字经济占GDP比重 > 50%
- 数据要素市场化
- AI赋能实体经济

**天眼的叙事**：
> "天眼是数据要素在商业决策领域的最佳实践——
> 不碰原始数据，100%合成数据，零隐私风险，最大决策价值。"

**行动**：
- 申请"数据要素X典型案例"
- 对接地方政府的数字化转型项目
- 争取纳入"国家AI创新应用先导区"

### 杠杆2：成本颠覆 — 100倍效率差

```
传统调研：
  预算：¥100,000+
  周期：3-6个月
  样本：200-500人
  准确率：~60%

天眼仿真：
  预算：¥5,000-20,000
  周期：72小时
  样本：10,000-100,000人（合成）
  准确率：待验证（目标75%+）
```

**关键洞察**：不要说服客户"AI更准"，要让他们意识到"100倍便宜+100倍快"本身就值得切换。即使准确率一样，成本和速度优势足以颠覆市场。

### 杠杆3：网络效应 — 合成人群的飞轮

```
更多客户使用 → 更多场景验证 → 合成人群更精准
    ↑                                    ↓
    ←←←← 更多客户因为精准而加入 ←←←←←←←
```

**核心资产不是代码，是合成人群库本身**。
每一个客户的使用都在训练更精准的合成人群。

### 杠杆4：生态杠杆 — 不卖预测，卖平台

**阶段1（0-6月）**：卖预测结果（天眼直接服务客户）
**阶段2（6-18月）**：卖预测能力（天眼平台+行业模板）
**阶段3（18月+）**：卖预测生态（开发者在天眼平台上构建垂直应用）

```
类比：
AWS不卖服务器，卖算力平台
天眼不卖调研报告，卖决策计算平台
```

## 四、竞争策略：建立护城河

### 第一道护城河：合成人群库

```
天眼的核心资产：
├── 城市人群画像库（一线→五线，全覆盖）
├── 消费行为模型库（200+品类）
├── 渠道偏好数据库（5大电商平台）
├── 社交传播模型库（KOL/种草/私域）
└── 政策影响模型库（医保/税收/产业）
```

**护城河来源**：这是用时间+客户积累的数据飞轮，不是竞争对手砸钱能追上的。

### 第二道护城河：合规壁垒

```
天眼合规架构：
├── PIPL合规（100%合成数据）
├── 数安法合规（数据不出境）
├── 算法备案（推荐算法合规）
├── 等保2.0（安全等级保护）
└── 内容审查（政治红线自动拦截）
```

**护城河来源**：合规架构是"嵌入骨头里的"，不是事后加的。竞品要复制，需要从头重建。

### 第三道护城河：行业Know-How

```
行业专属模板：
├── 医药：集采影响/新药上市/患者分群
├── 快消：新品测试/渠道优化/促销策略
├── 汽车：定价策略/配置偏好/区域差异
├── 金融：产品设计/风险定价/客户分群
├── 地产：区域市场/价格弹性/政策影响
└── 教育：课程设计/定价策略/渠道匹配
```

**护城河来源**：每个行业的仿真模型都需要真实的业务理解，不是纯AI能搞定的。

### 第四道护城河：生态锁定

```
天眼生态：
├── 数据供应商（统计局/电商/社交平台）
├── 渠道合作伙伴（咨询公司/广告公司）
├── 开发者（在天眼平台上构建应用）
└── 行业协会（标准制定话语权）
```

## 五、进入策略：三步走

### Step 1：标杆客户（0-6月）

**目标**：拿下3-5个行业标杆客户，建立案例

**策略**：
1. **医药先行**（你最懂的领域）
   - 用MediSlim/MediPharma的客户关系切入
   - 首个案例：GLP-1产品上市预测
   - 卖点：替代6个月的市场调研，72小时出结果

2. **快消跟进**
   - 找一个国货美妆/新消费品牌做POC
   - 免费或低价做，换案例和口碑

3. **建立案例库**
   - 每个案例都写成详细case study
   - 用真实数据证明天眼的预测准确率

### Step 2：平台化（6-18月）

**目标**：从"卖服务"到"卖平台"

**策略**：
1. **行业模板产品化**
   - 把标杆案例抽象成行业模板
   - 新客户可以"开箱即用"

2. **自助化平台**
   - 客户自己上传场景 → 自动匹配合成人群 → 自动仿真 → 自动出报告
   - 边际成本趋零

3. **渠道合作**
   - 与奥美/蓝标等广告公司合作
   - 与麦肯锡/贝恩等咨询公司合作
   - 他们用天眼做交付，天眼收平台费

### Step 3：生态化（18月+）

**目标**：从"平台"到"生态"

**策略**：
1. **开放API**
   - 第三方开发者可以在天眼平台上构建垂直应用
   - 天眼提供合成人群+仿真引擎，开发者提供行业Know-How

2. **数据联盟**
   - 与数据供应商建立合作（脱敏数据输入，合成人群输出）
   - 数据不出库，模型在天眼跑

3. **标准制定**
   - 推动"合成人群仿真"的行业标准
   - 谁定标准，谁就是品类定义者
