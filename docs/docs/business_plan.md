# 👁️ 天眼 (Tianyan) — 商业计划书

## 一、项目概述

天眼 = Aaru的中国版 —— 砍掉政治预测，深耕商业模拟，合规先行。

**核心理念**：用AI合成人群替代传统调研，72小时完成6个月的市场洞察。

## 二、Aaru模式拆解

### Aaru做什么
- 用多Agent AI模拟生成合成人群（替代真人调研）
- 三产品线：Lumen(商业) + Seraph(政务) + Dynamo(政治)
- 客户：McDonald's, Accenture, EY, IPG
- 估值：$1B（2025 Series A）

### Aaru为什么成功
1. **成本碾压**：传统调研6个月，Aaru几小时
2. **覆盖面广**：能触及"难触达人群"（高净值、偏远选民）
3. **无偏见**：基于行为数据而非自我报告
4. **可规模化**：同一天跑3600+受访者的30个市场

## 四、技术架构

```
数据层（公开统计数据）
    ↓
合成人群工厂（100%合成，PIPL合规）
    ↓
多Agent模拟引擎（MIMO API推理）
    ↓
三眼产品矩阵（消费眼/政策眼/市场眼）
    ↓
合规层（PIPL+数安法+算法备案+审计日志）
```

### 技术优势
1. **MIMO API无限额度**：零边际成本推理（Aaru按量付费）
2. **Harness架构**：可复用的模拟Harness设计
3. **合规内置**：不是事后加合规，是从架构层内置

## 七、合规框架

### 法律红线
1. PIPL：不使用真实个人信息，100%合成数据
2. 数据安全法：数据不出境，服务器在国内
3. 算法推荐管理规定：算法备案
4. 网络安全法：等保2.0

### 内容红线
- ❌ 不做政治预测
- ❌ 不做社会动荡模拟
- ❌ 不做敏感政策评价
- ✅ 经济/民生/产业/教育/医疗政策评估

## 九、Harness理论应用

天眼本身就是Harness理论的极致体现：

```
天眼 Harness = 合成人群工厂 + 多Agent模拟引擎 + 场景编排 + 合规层 + 结果验证
```

- 合成人群 = 上下文管理
- 多Agent模拟 = 工具链编排
- 合规层 = 失败恢复（违规时自动拦截）
- 结果验证 = 与真实数据交叉验证

**护城河**：不是AI模型，是整套模拟架构的Harness设计。
